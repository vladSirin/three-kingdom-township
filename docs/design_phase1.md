# 三国小镇 (Three Kingdoms Township) - Phase 1 进展与设计文档

## 1. 当前开发进展 (截至 Phase 1)

目前游戏的核心底层循环已经完成了向 **PDX（Paradox-style）机制** 的跃迁，旨在通过动态修饰器和隐性状态替代简单的硬编码数值增减，以提供更加深度的沙盒与涌现式体验。

### 1.1 全局修正枢纽 (Effect System)
- **解耦数值逻辑**：资源的产出与消耗不再直接由建筑等级计算，而是全部交由 `EffectSystem` 接管。
- **机制引入**：
  - `STATES_DICT` (临时环境状态)：如“饥荒”、“安居乐业”、“待宰肥羊”等。这些状态会在岁末结算时，根据城镇的各项资源情况（比如拥有过多的财富但武力极低）自动判定进入。
  - `TRAITS_DICT` (主公特质)：如“仁义”、“暴虐”等。为主公附加的永久性标签，同时包含正反两面的影响（“盲盒”哲学设计）。
- 将所有状态和特质的修饰项（Modifiers）组合在一起，统一影响最终资源的增减与事件概率的权重。

### 1.2 UX/UI 革新
- **主公档案面板 (Profile Modal)**：在页面右上角新增主公档案按钮。点击打开带有毛玻璃效果和高雅暗黑风格的弹窗，显示角色的抽象“特质标签”。具体数值效果对玩家隐藏，鼓励基于角色扮演（Roleplay）来推动游戏。
- **状态浮窗系统 (Toast UI)**：弃用了原本只在弹幕中提示重要状态变更的做法，改为基于页面顶部中心下放的醒目横向悬浮提示窗 (`showToast`)，让玩家不错过宏观的负面或正面状态提醒。

## 2. 危机预警与缓冲机制设计方案 (Game Over Rework)

针对过去版本中存在的“猝死（硬性数值一旦触底 / 触顶直接导致屏幕变黑 Game Over）”问题，全面设计并实装了“危机缓冲”方案。

### 2.1 软着陆缓冲池 (Soft Game Over)
- 废弃了“一碰到 0 或 100 就死”的逻辑。
- 取而代之，当某项资源（例如“粮草”）≤ 0，城镇将会获得特定高危状态（如：**[断粮崩溃]**），并附带 **3回合倒计时** 的倒数限制。
- 处于倒计时危机中，每回合除了该资源无法增长外，还会牵连引发额外的致命惩罚（例如极大扣除民心、流出人口）。
- **抢救解危**：玩家若能在3回合之内，通过特定的事件或产出，将崩盘资源重新抬高至安全线之上（≥ 1点），则该危机状态会自动解除。
- 只有当3回合危机倒计时归零且数值仍未得救，才判定最终“身死家灭”。

### 2.2 视觉焦虑感重塑 (Visual Alarms)
- **红色心跳脉冲 (Pulse Animations)**：当顶部资源栏位陷入 ≤ 20% 原上限（或 >= 90%高危上限）时，对应的资源槽会从普通状态变红并保持强烈的高频发光闪烁，在UI层面营造强烈的局促与危机感。
- **频斩提示**：陷入危机和获得危机相关负面事件时，不仅由 `showToast` 弹出最高级别的刺眼红色系统警报，同时触发 `screenShake` 实现物理全屏视觉震动，唤起玩家高度警觉。

### 2.3 强制灾祸事件池 (Disaster Interception)
- 城镇处于软着陆死亡倒计时期间，每回合将不再触发普通内政或民生事件。
- `events.js` 实装了专属拦截表 `window.DISASTER_EVENTS`。系统会根据当前倒计时引发专门对应的**“极危勒索卡片”**：
  - 比如断粮期间必定触发“饥民暴动”。
  - 选项不再具备普通收益，必定逼迫玩家进行极其残酷的抉择（如：散尽其余财产和名声强行高价购买黑市粮食，或者武力血洗屠镇以削减粮食消耗）。
  - 这些断臂求生事件旨在给将死的城镇增加最后的翻盘戏剧性，增强游戏决策的深度与残酷感。

### 2.4 上限的高风险博弈 (High Value Crises)
- 解决了之前的“囤积过百”反而导致冤死惩罚的安全漏洞。
- 修改机制为：当资源达到当前存储建筑所能容纳的 90% 极值时（且不限制绝对常数100），给城镇挂上属于富贵病状态（如 **[树大招风]**、“**[囤积招患]**）。
- 富贵病同样具备倒计时，并在此倒计时内引入强力打劫、军阀强借粮草的高危、高损耗勒索事件，进一步将数值封顶变成了高风险高收益的惊险博弈状态。

## 3. 下一步计划 (Next Steps)
现有的框架升级和 UI 面板已全部打通。接下来的首要目标是：
- **本地存档机制 (Save & Load Storage)**：利用浏览器的 `localStorage` 为网页版完成自动存档与读档支持，且反序列化过程必将全面兼容已引入的 `GameState.activeStates` 与 `GameState.traits`。
